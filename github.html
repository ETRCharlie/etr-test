GitStuff

A respository (repo) is a project. A branch is a small offshoot of the main repo that will likely be deleted after it is committed back to the repository. A fork is a separate copy of that repo which will likely live on its own.

The basic usage is: Create (or get access to) a repo at GitHub, copy it to your local machine ("clone"), make your changes locally, let GitHub know you have changes ready to move into production ("commit"), push them into production ("pull").

1. download and install Github Desktop app
2. download and install command-line app

GUI Usage

1. Log into your GitHub account in your browser. Click the "start a project" button. On the next page, give your new project ("repository") a name. Put a check in the "initialize this repository with a README" box, then click the green "create repository" button.

2. In the next page, click the "set up in desktop" button. This will prompt you to select an app, select "GitHub Desktop.exe", click the "remember my choice box" and click "open link".

3. The GitHub app will start, and you'll get a popup window asking to confirm the local file path. Click the "clone" button. This should show you the local repo in the GUI with no changed files.

4. Find the GitHub folders you specified in step 3, make an edit to any file, and this should prompt that filename to appear in the GUI under the "current repository" pane. If not, click in the white area of that pane and it should appear.

5. When your changes are ready to commit to the repo at GitHub, make sure there is a checkmark next to the filename in the "current repository" pane. Add a summary at the bottom of that pane to state what you changed and click the blue "commit to master" button.

Command-Line Usage

GitStuff Commands

git add *
adds all the local files to the git repo

git status
shows you the state of the git repo

git add file.name
stages that file for committing

git commit -m "message with description of changes"
commits that file to the repo

git commit -a -m "adds and commits all files"
adds ALL files and commits them (combines previous two commands)

git push
moves the committed files up to GitHub to update the repo

git push --set-upstream origin BranchName

Other GitStuff

git log

git checkout -b NewBranchName

git branch

git checkout master
returns you to the master repo
