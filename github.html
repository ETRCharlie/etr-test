GitStuff

A respository (repo) is a project. A branch is a small offshoot of the main repo that will likely be deleted after it is committed back to the repository. A fork is a separate copy of that repo which will likely live on its own.

The basic usage is: Create (or get access to) a repo at GitHub, copy it to your local machine ("clone"), make your changes locally, let GitHub know you have changes ready to move into production ("commit"), push them into production ("pull").

First, download and install Github Desktop app, then download and install command-line app. Install both as your own user, now as admin.

GitStuff Commands

git add *
adds all the local files to the git repo

git status
shows you the state of the git repo

git add file.name
stages that file for committing

git commit -m "message with description of changes"
commits that file to the repo

git commit -a -m "adds and commits all files"
adds ALL files and commits them (combines previous two commands)

git push
moves the committed files up to GitHub to update the repo

git push --set-upstream origin BranchName

Other GitStuff

git log
shows you the history (q to quit)

git checkout -b NewBranchName
branches the repo

git branch
tells you which branch you're working on

git checkout master
returns you to the master repo
